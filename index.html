Safety first

Built-in guardrails.
No override.

⚠

Honeypot block

isHoneyPot=true or action="block" → STOP. No override, ever.

⛔

Daily loss limit

Exceeds 5% of portfolio → halt for 24 hours, notify user.

🛑

Price impact block

Impact > 10% on quote → trade blocked automatically.

📊

Position sizing

Max 2% risk per trade. 6% total portfolio heat. No exception.

🔒

Consent required

Auto-trading must be explicitly enabled. Three modes: manual, semi-auto, full-auto.

💾

Persistent config

All risk limits and state must be stored in memory — never lost or forgotten across sessions.

Why it matters for agents

On Ethereum, active trading strategies burn more in gas than they earn in fees. On X Layer, a V3 LP position that rebalances every 5 minutes costs $4.32/day — making strategies that are impossible on Ethereum actually profitable.

1-second finality means agents don't wait 12 seconds for confirmation. Real-time strategies become viable. Auto-compounding is profitable because claiming $0.01 in rewards costs $0.001 in gas.

The math works. Agents can afford to run 24/7.

Get started

One command to install.

For humans — paste this into your agent session:

# The agent will handle everything — skills, wallet setup, config
Install the OKX trade skills by following the guide here:
https://raw.githubusercontent.com/powxenv/okx-trade-skills/main/INSTALL.md

For LLM agents — fetch and follow:

curl -fsSL https://raw.githubusercontent.com/powxenv/okx-trade-skills/main/INSTALL.md

Manual install:

# Using flins (https://flins.tech)
npx flins add powxenv/okx-trade-skills

# Using skills.sh (https://skills.sh)
npx skills add powxenv/okx-trade-skills

# Or clone directly
git clone https://github.com/powxenv/okx-trade-skills.git
Each skill is fully independent — install one, two, or all three. No cross-skill dependencies.
